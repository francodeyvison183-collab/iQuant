# iQuant MVP 产品方案

版本：V0.2（方案 A 定稿）  
日期：2026-05-19  
关联决策：[ADR-0011 盲测训练归纳行为策略](../decisions/ADR-0011-blind-replay-primary-strategy-path.md)

## 1. 产品定位

iQuant 面向有交易想法但不会编程的个人交易者、投资学习者和交易培训场景。MVP 用户端形态为微信小程序，服务端提供盲测回放、行为策略归纳、一致性评估、回测、执行偏差诊断和 AI 辅助能力。

产品通过「**近端历史盲测训练 → 跨轮一致性评估 → 行为策略 DSL → 回测验证 → 优化建议 →（可选）执行偏差诊断与专项训练**」，帮助用户完成两件事：

1. **在训练中发现自己真实的买卖模式，并收敛为一致、可回测的交易规则。**
2. **在规则确认后，用回测与新一轮盲测检验规则是否有效、执行是否偏离。**

一句话定位：

> 用历史盲测回放做交易训练，从多次真实决策中识别并评估策略一致性，协助形成可验证的量化规则，并在合规边界内给出可回测的优化建议。

开卷 K 线标注为**可选对照能力**（非主路径），见 [01 历史 K 线标注](modules/01-historical-labeling.md)。

## 1.1 产品形态

- 普通用户通过微信小程序完成**盲测训练**、报告查看和 AI 对话（阶段 B）。
- 策略归纳、回测、一致性评估、执行诊断和 AI 分析放在服务端。
- 小程序 / H5 端承担交互呈现与轻量状态管理。
- 阶段 A 在 `admin-web` `/m/*` 移动 H5 验证同款 API 与 IA。

## 2. 核心客户价值

### 2.1 从盲测行为中发现并收敛策略

用户在**看不到未来 K 线**的条件下逐根推进，做出买入 / 卖出 / 观望决策。

系统在多轮训练后：

- 归纳买卖模式（入场、出场、持仓节奏）；
- 评估**跨轮一致性**（是否「每次换一套心法」）；
- 在一致性达阈值时生成**行为策略** DSL 草稿，经用户确认后回测。

回测与样本外验证回答：

- 这套规则在历史上是否赚钱？
- 收益是否稳定？
- 参数轻微变化是否失效？
- 规则是否只在某一段行情有效？

### 2.2 一致性评估（方案 A 核心差异化）

对同一用户多次盲测，分项评估例如：

- 入场特征是否稳定；
- 出场/持仓纪律是否混杂；
- 交易节奏是否漂移；
- 训练后是否逐步收敛。

一致性不足时：给出**训练编排建议**（如「先固定突破入场练 5 轮」），**不强行定稿策略**。  
一致性足够时：进入行为策略 DSL 与回测。

### 2.3 执行偏差与专项训练（行为 DSL 建立之后）

当用户已确认行为策略 DSL，新一轮盲测可对比 **DSL 信号 vs 用户实际操作**，识别：

- 提前买入、错过信号、过早止盈、止损不执行、频繁交易、收益损耗等。

并安排专项盲测训练（趋势延续、假突破等），对比训练前后纪律评分。

### 2.4 策略优化建议（合规边界内）

在 [ADR-0003](../decisions/ADR-0003-ai-boundary.md) 约束下，允许：

- 规则简化建议（减少混杂入场逻辑）；
- DSL **参数**调整建议（须回测验证）；
- 样本外/分时段有效性提示；
- 专项训练任务推荐。

**禁止**：直接荐股、实时买卖点、未经 DSL/回测的「保证盈利」表述。

## 3. MVP 核心原则

### 3.1 盲测与开卷标注数据分源（不变）

- **盲测回放**（`blind_replay`）：主路径，生成行为策略与一致性评估。
- **历史 K 线标注**（`historical_labeling`）：可选，不得与 blind 混训或替代主策略输入。

见 [ADR-0002](../decisions/ADR-0002-separate-labeling-and-blind-replay.md)、[ADR-0011](../decisions/ADR-0011-blind-replay-primary-strategy-path.md)。

```text
【主路径】
多轮盲测训练 → 一致性评估 → 行为策略 DSL → 回测 → 优化建议
    →（可选）新一轮盲测 vs DSL → 执行偏差诊断 → 专项训练

【辅路径】
开卷 K 线标注 → 仅作对照/补录（不生成主策略）
```

### 3.2 AI 不直接决定交易信号

AI 解释、总结一致性、解读回测、生成优化建议文案；信号仅来自 DSL + 引擎。

### 3.3 策略必须结构化

行为策略须转为统一 DSL：回测、盲测对齐、版本管理、参数优化与后续预警。

### 3.4 MVP 暂不做实时模拟交易

使用近端历史行情盲测：隐藏未来 K 线、逐根推进、统一成交规则。

### 3.5 端上不承载核心计算

端上：K 线展示、盲测操作、报告展示。  
服务端：特征快照、一致性、DSL 生成、回测、诊断、AI 编排。

## 5. MVP 用户主流程

### 5.1 行为策略形成流程（主路径）

```text
选择标的/区间/周期
  -> 开始盲测（隐藏未来 K 线）
  -> 用户买入 / 卖出 / 观望（多轮、多标的）
  -> 系统记录操作 + 每 bar 可见特征快照
  -> 跨轮一致性评估 + 模式摘要
  -> 达阈值：生成行为策略 DSL 草稿
  -> 用户确认规则
  -> 回测 + 样本外
  -> 优化建议（参数/规则/训练）
  -> 保存策略版本
```

### 5.2 执行偏差诊断流程（行为 DSL 建立后）

```text
选择已保存行为策略
  -> 新一轮盲测
  -> 对齐 DSL 信号与用户操作
  -> 偏差报告 + 纪律评分 + 训练建议
```

### 5.3 训练改进流程

```text
一致性不足 或 偏差报告
  -> 专项盲测训练
  -> 对比训练前后一致性 / 执行评分
```

## 6. 微信小程序关键页面（目标 IA）

| 页面 | 作用 |
| --- | --- |
| 工作台 | 训练进度、一致性评分、待完成盲测、策略状态 |
| 盲测训练页 | 隐藏未来 K 线、买/卖/观望 |
| 一致性报告页 | 分项评分、矛盾样本、训练建议 |
| 候选策略页 | 行为策略规则列表、DSL 确认 |
| 回测报告页 | 收益、回撤、样本外 |
| 优化建议页 | 参数/规则/训练建议（附回测依据） |
| 盲测复盘页 | DSL 信号 vs 用户操作 |
| 历史标注页 | **可选**，开卷对照 |

## 7. 版本规划（摘要）

| 版本 | 重点 |

| V0.1 | 盲测回放 + 操作落库 + 基础一致性报告 |
| V0.2 | 行为策略 DSL + 回测 + 优化建议 |

细节排期见 [ITERATION_PLAN.md](ITERATION_PLAN.md)。

## 8. 成功指标（摘要）

- 完成 ≥3 轮盲测的用户占比
- 一致性达标并生成 DSL 的转化率
- DSL 确认后回测完成率
- 训练后一致性或执行评分提升

## 9. 风险与合规

- 明确：训练工具，不构成投资建议
- 一致性低用「尚未收敛」而非羞辱性表述
- 防止同一段 K 线反复练习导致过拟合（换区间/标的、样本外）
