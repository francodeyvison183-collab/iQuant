# 03 回测验证模块

## 1. 模块定位

回测验证模块用于验证理想策略是否具备历史盈利能力和稳定性。

它回答的问题是：

> 如果用户严格执行这套理想策略，历史上是否有效？

## 2. 用户价值

- 判断理想策略是否真的有统计优势。
- 避免只凭复盘感觉确认策略。
- 提醒过拟合和不可承受风险。

## 3. 核心边界

本模块负责：

- 执行策略 DSL。
- 生成历史交易记录。
- 计算收益和风险指标。
- 做样本内、样本外和参数敏感性验证。

本模块不负责：

- 生成策略。
- 盲测回放。
- 解释用户执行偏差。

## 4. 默认回测规则

MVP 固定以下规则，降低歧义：

- 只做多。
- 同一标的同一时间只持有一笔仓位。
- 信号在当前 K 线收盘后生成。
- 下一根 K 线开盘价成交。
- 默认满仓或固定仓位。
- 默认手续费 0.05%。
- 默认滑点 0.05%。
- 不允许未来函数。

## 5. 验证维度

| 维度 | 说明 |
| --- | --- |
| 样本内表现 | 策略在标注相关区间表现如何 |
| 样本外表现 | 策略在未参与生成的区间表现如何 |
| 行情阶段表现 | 上涨、下跌、震荡阶段分别表现 |
| 参数敏感性 | 参数轻微变化后是否失效 |
| 收益分布 | 是否依赖少数大盈利交易 |
| 风险承受 | 最大回撤和连续亏损是否可接受 |

## 6. 核心指标

- 总收益。
- 年化收益。
- 最大回撤。
- 胜率。
- 盈亏比。
- 交易次数。
- 平均持仓天数。
- 最大连续亏损。
- 样本外收益。
- 参数稳定性评分。

## 7. 输入与输出

输入：

- 策略 DSL。
- 历史 K 线数据。
- 成交规则配置。
- 费用和滑点配置。

输出：

- 回测报告。
- 交易明细。
- 资金曲线。
- 回撤曲线。
- 样本内 / 样本外对比。
- 风险提示。

## 8. 验收标准

- 策略 DSL 能稳定执行。
- 所有交易记录可追溯到具体 K 线。
- 能输出核心收益和风险指标。
- 能明确提示过拟合风险。
