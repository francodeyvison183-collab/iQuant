# 08 策略库与预警准备模块

## 1. 模块定位

策略库与预警准备模块用于保存理想策略、回测报告、执行诊断报告和策略版本。

MVP 阶段不一定上线真实行情预警，但必须提前设计好策略保存和信号记录结构。

## 2. 用户价值

- 用户可以沉淀自己的策略资产。
- 用户可以对比不同版本策略的表现。
- 后续接入行情接口后，可以平滑升级到信号预警。

## 3. 核心边界

本模块负责：

- 策略保存。
- 策略版本管理。
- 回测报告归档。
- 执行诊断报告归档。
- 预警配置模型预留。

本模块不负责：

- 实时行情接入。
- 实盘交易。
- 策略生成。
- 回测计算。

## 4. 策略版本

每次策略参数变化都应形成新版本。

保存内容：

- 策略名称。
- 标的。
- 周期。
- 策略 DSL。
- 参数。
- 创建来源。
- 回测报告 ID。
- 执行诊断报告 ID。
- 创建时间。

## 5. 预警准备

即使 MVP 暂不做实时预警，也应预留：

- 策略启用状态。
- 监控标的。
- 监控周期。
- 信号触发条件。
- 信号记录表。
- 通知渠道配置。

后续真实行情接口成熟后，可接入：

- 每日收盘后扫描。
- 实时或准实时扫描。
- 站内通知。
- 邮件或 Webhook。

## 6. 输入与输出

输入：

- 策略 DSL。
- 回测报告。
- 执行诊断报告。
- 用户保存操作。

输出：

- 策略版本。
- 策略库列表。
- 策略详情。
- 预警配置草案。

## 7. 验收标准

- 用户可以保存策略。
- 用户可以查看历史版本。
- 每个策略版本能关联回测报告和执行诊断报告。
- 预警字段结构不阻碍后续扩展。
