# 04 近端历史盲测回放模块

## 1. 模块定位

近端历史盲测回放模块使用最近历史行情，但隐藏未来 K 线，让用户在未知后续走势的状态下做买卖决策。

该模块用于记录用户实际执行行为，不用于生成理想策略。

## 2. 用户价值

- 降低历史复盘的后视偏差。
- 让用户体验更接近真实交易的决策压力。
- 记录用户是否能执行自己的理想策略。

## 3. 核心边界

本模块负责：

- 隐藏未来 K 线。
- 控制 K 线逐根或按步长播放。
- 处理用户买入、卖出、观望操作。
- 记录用户操作时的策略信号和行情快照。
- 生成盲测交易记录。

本模块不负责：

- 生成理想策略。
- 判断策略盈利能力。
- 输出最终偏差结论。

## 4. 盲测流程

```text
选择已保存理想策略
  -> 选择盲测标的和区间
  -> 系统显示当前及之前 K 线
  -> 用户逐步推进行情
  -> 用户买入 / 卖出 / 观望
  -> 系统记录当前策略信号与用户操作
  -> 完成盲测
  -> 提交给执行偏差诊断模块
```

## 5. 功能范围

- K 线回放。
- 隐藏未来行情。
- 买入、卖出、观望。
- 快进 N 根 K 线。
- 当前持仓展示。
- 当前浮盈浮亏展示。
- 操作记录。
- 用户交易理由标签。
- 信心程度记录。

## 6. 默认成交规则

- 当前 K 线收盘后决策。
- 下一根 K 线开盘成交。
- 手续费 0.05%。
- 滑点 0.05%。
- 只做多。
- 同一标的同一时间只持有一笔仓位。

```json
  "confidence": "medium"

```

## 8. 验收标准

- 用户无法看到未来 K 线。
- 用户每次操作都能记录当时策略信号。
- 盲测记录能被执行偏差诊断模块读取。
- 交易成交规则与回测模块保持一致。
