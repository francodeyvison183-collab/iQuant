# 04 近端历史盲测回放模块

版本：V0.2（方案 A 主路径）  
关联：[ADR-0011](../../decisions/ADR-0011-blind-replay-primary-strategy-path.md)

## 1. 模块定位

近端历史盲测回放是 iQuant **产品主路径**的核心交互：在最近历史行情上**隐藏未来 K 线**，让用户在未知后势的条件下做出买入 / 卖出 / 观望决策。

系统记录：

- 用户操作序列；
- 每个操作时刻的**可见行情特征快照**（供模式归纳与一致性评估）；
- 模拟持仓与成交结果。

该模块产出供 **一致性评估**、**行为策略生成** 消费；在行为 DSL 确认后，新一轮盲测还可供 **执行偏差诊断** 消费。

## 2. 用户价值

- 降低后视偏差，接近真实决策压力。
- 多轮训练后可发现「实际如何买卖」，而非「事后希望如何买卖」。
- 为「策略一致性」提供可量化依据。

## 3. 核心边界

### 3.1 本模块负责

- 隐藏未来 K 线（**服务端裁剪**，不可仅依赖前端）。
- 控制 K 线逐根或按步长播放。
- 买入、卖出、观望及统一成交规则。
- 操作 + 特征快照落库（`source=blind_replay`）。
- 会话级进度与模拟持仓状态。

### 3.2 本模块不负责

- 跨轮一致性评分（见迭代 2 用例，可同服务不同用例）。
- 生成最终 DSL（行为策略生成模块）。
- 回测计算。
- 直接输出买卖建议或荐股。

## 4. 盲测流程（主路径）

```text
选择标的、周期、近端区间
  -> 创建 blind_session
  -> 逐 bar 推进（仅返回 visible_until 之前 K 线）
  -> 用户 buy / sell / hold
  -> 记录 blind_action + features_snapshot
  -> 结束会话
  ->（多轮后）一致性评估 / 模式摘要
  ->（达标后）行为策略 DSL
```

### 4.1 与行为 DSL 建立后的流程

```text
选择已保存行为策略
  -> 新一轮 blind_session
  -> 每 bar 记录 strategy_signal + user_action
  -> 结束 -> 执行偏差诊断
```

## 5. 功能范围

- K 线回放与可见窗口控制。
- 买入、卖出、观望；快进 N 根（P1）。
- 持仓、现金、浮盈浮亏展示。
- 操作理由标签、信心（P1）。
- 多轮会话列表与回看（只读）。

## 6. 默认成交规则

- 当前 K 线收盘后决策。
- 下一根 K 线开盘成交。
- 手续费 0.05%；滑点 0.05%。
- 只做多；单标的单仓位。

## 7. 核心数据

```json
{
  "id": "blind_action_001",
  "source": "blind_replay",
  "session_id": "uuid",
  "full_code": "sz300750",
  "period": "day",
  "visible_until": "2025-11-12T00:00:00Z",
  "user_action": "buy",
  "features_snapshot": { "ma20_dist": 0.02, "volume_ratio": 1.1 },
  "strategy_signal": null,
  "user_reason": ["breakout"],
  "confidence": "medium"
}
```

`strategy_signal` 在「对照已确认 DSL」阶段填充；归纳阶段为空。

## 8. 依赖关系

**依赖**：行情数据服务、K 线组件、（后期）策略信号引擎。

**被依赖**：一致性评估、行为策略生成、执行偏差诊断、盲测训练编排。

## 9. 验收标准

- [ ] 任意接口无法获取未来 bar。
- [ ] 每步操作含可解释特征快照。
- [ ] 与 `label_*` 数据/API 隔离。
- [ ] 完成 ≥3 轮后可触发一致性报告（与迭代 2 联调）。
