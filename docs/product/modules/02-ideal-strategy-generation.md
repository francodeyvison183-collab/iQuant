# 02 理想策略生成模块

## 1. 模块定位

理想策略生成模块根据历史 K 线标注样本，生成可解释、可回测、可版本化的候选理想策略。

该模块只处理历史标注样本，不混入盲测回放交易样本。

## 2. 用户价值

- 将用户的主观标注转成明确交易规则。
- 降低非程序化用户构建策略的门槛。
- 让用户获得可解释的候选策略，而不是黑盒模型。

## 3. 核心边界

本模块负责：

- 从标注点提取技术特征。
- 从内置策略模板中匹配候选策略。
- 生成策略 DSL。
- 生成策略自然语言摘要所需的结构化信息。

本模块不负责：

- 回测计算。
- 执行偏差诊断。
- 盲测回放。
- AI 直接生成买卖信号。

## 4. 生成方式

MVP 阶段采用：

```text
技术特征提取
  -> 模板匹配
  -> 参数搜索
  -> 策略 DSL 生成
  -> 候选策略排序
```

暂不采用复杂黑盒模型。

## 5. 内置策略模板

| 模板 | 买入逻辑 | 卖出逻辑 |
| --- | --- | --- |
| 均线突破 | 收盘价上穿 MA20 / MA30 | 跌破 MA10 / MA20 |
| 放量突破 | 突破前高且成交量放大 | 跌破短期均线或止盈止损 |
| 均线回踩 | 趋势向上，回踩关键均线后转强 | 跌破趋势均线 |
| MACD 转强 | MACD 金叉或柱体增强 | MACD 转弱或固定风控 |
| 超跌反弹 | RSI 低位回升且收阳 | RSI 高位回落或固定止盈 |

## 6. 策略 DSL 示例

```json
{
  "name": "放量突破 MA20",
  "entry": {
    "all": [
      { "cross_above": ["close", "ma20"] },
      { ">": ["volume_ratio_5", 1.3] }
    ]
  },
  "exit": {
    "any": [
      { "cross_below": ["close", "ma10"] },
      { ">=": ["profit_pct", 0.1] },
      { "<=": ["profit_pct", -0.05] }
    ]
  },
  "position": {
    "direction": "long",
    "size": "100%",
    "max_holding_bars": 30
  }
}
```

## 7. 输入与输出

输入：

- 历史标注样本。
- K 线数据。
- 指标数据。
- 策略模板库。

输出：

- 3 到 5 个候选理想策略。
- 策略 DSL。
- 命中标注样本情况。
- 策略解释素材。

## 8. 候选策略排序

建议综合考虑：

- 标注匹配度。
- 规则复杂度。
- 参数是否稳定。
- 交易次数是否合理。
- 是否容易解释。

## 9. 验收标准

- 系统能基于至少 10 对历史标注生成候选策略。
- 每个候选策略必须有明确买入规则和卖出规则。
- 每个候选策略必须能转成 DSL。
- 候选策略可提交给回测验证模块。
