# 01 历史 K 线标注模块（辅助）

> **产品定位（2026-05-19）**：本模块**不是**主路径。主路径为 [04 盲测回放](04-blind-replay.md) → 一致性 → 行为策略，见 [ADR-0011](../../decisions/ADR-0011-blind-replay-primary-strategy-path.md)。  
> 开卷标注用于**对照、补录、运营**，不得替代 blind 样本生成主策略。

版本：V0.2  
关联：[ADR-0002](../../decisions/ADR-0002-separate-labeling-and-blind-replay.md)

## 1. 模块定位（辅助）

用户在**完整历史 K 线（开卷）**上点选理想买入、卖出，表达「事后认为应该如何交易」。

与盲测（闭卷、类实战）相对，用于：

- 与用户自述对照；
- 运营补录样本；
- 未来 P2：对比「开卷目标」与「闭卷行为」差异。

**不**作为行为策略 DSL 的默认输入。

## 2. 已实现能力（仓库现状）

| 能力 | 状态 |
| --- | --- |
| `label_session` / `label_pair` | ✅ |
| 批次队列标注、轮次总结 | ✅ |
| API `/api/v1/labels/*` | ✅ |
| admin-web H5 标注页 | ✅（入口建议标「高级」） |

工程说明：[annotation 模块](../../architecture/modules/annotation.md)

## 3. 核心边界

- 展示 K 线、买卖配对、收益计算。
- 数据 `source=historical_labeling`（或与 blind 分表）。
- **禁止**与 blind 混训或写入同一混淆来源。

不负责：策略是否盈利、盲测交互、主路径一致性评估。

## 4. 后续（P2）

- 买/卖意图标签。
- 与行为策略对照报告。
- 不扩展为默认策略生成输入。

## 5. 验收标准（维护级）

- [ ] 与 blind API/表无混合入口。
- [ ] 文档与 H5 不宣称「标注即可生成主策略」。
