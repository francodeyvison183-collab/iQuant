# 05 执行偏差诊断模块

## 1. 模块定位

执行偏差诊断模块用于比较“理想策略信号”和“用户盲测实际操作”，找出用户真实交易行为与理想策略之间的差距。

这是产品的核心差异化模块。

## 2. 用户价值

- 用户能知道亏损或收益不足到底来自策略问题还是执行问题。
- 用户能看到具体行为偏差，而不是泛泛地被评价“心态不好”。
- 用户能知道下一步应该训练什么。

## 3. 核心边界

本模块负责：

- 对齐策略信号与用户操作。
- 计算执行偏差指标。
- 计算执行导致的收益损耗。
- 生成纪律评分。
- 给训练模块提供偏差类型。

本模块不负责：

- 生成理想策略。
- 做盲测交互。
- 提供 AI 文案解释。

## 4. 核心指标

| 指标 | 含义 |
| --- | --- |
| 信号执行率 | 策略出现买卖信号时用户是否执行 |
| 错过信号率 | 策略给出信号但用户没有操作 |
| 非策略入场率 | 策略未给买入信号时用户买入 |
| 买入提前/延迟 | 用户相对策略信号早买或晚买的 K 线数 |
| 卖出纪律 | 策略触发卖出时用户是否离场 |
| 止损执行率 | 触发止损条件后是否执行 |
| 过早止盈率 | 策略未触发卖出时用户提前卖出 |
| 收益损耗 | 用户盲测收益与策略同期理论收益的差值 |

## 5. 诊断输出示例

```text
策略同期收益：+18.4%
用户盲测收益：+6.1%
执行损耗：-12.3%

主要偏差：
1. 4 次买入提前于策略信号。
2. 3 次盈利交易过早卖出。
3. 2 次止损信号出现后未执行。
```

## 6. 纪律评分

建议按 100 分制输出：

- 总体执行分。
- 买入纪律。
- 卖出纪律。
- 止损纪律。
- 止盈纪律。
- 等待能力。

## 7. 输入与输出

输入：

- 理想策略 DSL。
- 理想策略在盲测区间的理论信号。
- 用户盲测操作记录。
- 盲测成交结果。

输出：

- 偏差指标。
- 收益损耗。
- 纪律评分。
- 偏差交易清单。
- 训练建议输入。

## 8. 验收标准

- 每次用户操作都能和策略信号对齐。
- 能明确区分策略问题与执行问题。
- 能输出主要偏差类型排序。
- 能为训练模块生成训练任务依据。
