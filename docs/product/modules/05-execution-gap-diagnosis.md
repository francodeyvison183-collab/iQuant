# 05 执行偏差诊断模块

版本：V0.2（方案 A）

## 1. 模块定位

本模块分两阶段（[ADR-0011](../../decisions/ADR-0011-blind-replay-primary-strategy-path.md)）：

| 阶段 | 条件 | 输出 |
| --- | --- | --- |
| **一致性诊断** | 尚无确认的行为 DSL | 跨轮 blind 一致性分项、模式摘要、训练建议 |
| **执行偏差诊断** | 已有行为 DSL | DSL 信号 vs 用户新一轮 blind 操作 |

二者均基于**可观测行为**，避免心理断言。

## 2. 用户价值

- 一致性不足时：知道「尚未收敛」，该练什么。
- DSL 建立后：区分策略问题 vs 执行问题（信号执行率、收益损耗等）。

## 3. 核心边界

负责：

- 多轮 blind 一致性评分与矛盾样本。
- DSL 信号与用户操作逐 bar 对齐。
- 纪律评分、收益损耗、偏差类型。
- 为训练模块提供任务依据。

不负责：

- 盲测 UI。
- 生成 DSL（行为策略生成模块）。
- AI 文案（AI 模块）。

## 4. 核心指标（执行偏差阶段）

| 指标 | 含义 |
| --- | --- |
| 信号执行率 | 有信号时是否执行 |
| 错过信号率 | 有信号未操作 |
| 非策略入场率 | 无买入信号却买入 |
| 买入提前/延迟 | 相对信号的 bar 偏移 |
| 过早止盈率 | 未触发卖出却卖出 |
| 收益损耗 | 用户收益 vs 策略理论收益差 |

## 5. 一致性指标（归纳阶段）

| 分项 | 含义 |
| --- | --- |
| 入场一致性 | 买入时特征分布是否稳定 |
| 出场一致性 | 持仓/止盈模式是否混杂 |
| 节奏一致性 | 频率、空仓习惯 |
| 跨轮趋势 | 训练后是否收敛 |

## 6. 依赖关系

**依赖**：盲测回放、行为策略 DSL（执行阶段）、行情。

**被依赖**：盲测训练编排、AI 解释。

## 7. 验收标准

- [ ] 无 DSL 时可输出一致性报告。
- [ ] 有 DSL 时偏差类型可人工核对。
- [ ] 不消费 `label_*` 生成主策略报告。
