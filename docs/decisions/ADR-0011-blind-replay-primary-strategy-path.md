# ADR-0011 主路径：盲测训练归纳行为策略（方案 A）

状态：已接受（2026-05-19）  
取代范围：产品主流程中对 [ADR-0002](ADR-0002-separate-labeling-and-blind-replay.md)「标注生成理想策略、盲测仅做执行诊断」的**优先级**表述；**数据分源与分表要求不变**。

## 背景

原 MVP 假设用户能通过开卷 K 线标注表达「理想策略」，再借盲测对比执行差距。实践中：

- 多数用户难以稳定自述交易规则；
- 开卷标注易引入后视偏差，与真实决策距离较远；
- 产品核心价值更接近「在训练中发现自己**实际**如何买卖，并收敛为**一致、可回测**的规则」。

## 决策

采用 **方案 A** 作为产品主路径：

```text
多轮近端历史盲测回放（隐藏未来 K 线）
  -> 记录买/卖/观望 + 每 bar 可见行情特征快照
  -> 跨轮一致性评估（入场/出场/节奏等分项）
  -> 一致性达阈值后，归纳「行为策略」候选 DSL（用户确认）
  -> 回测与样本外验证
  -> 规则/参数/训练类优化建议（经 DSL patch + 回测，或专项盲测）
```

### 仍遵守的边界

1. **数据分源**（延续 ADR-0002）：`historical_labeling` 与 `blind_replay` **分表、分 API、禁止混训**；策略归纳**只消费** `blind_replay` 及确认后的行为 DSL。
2. **信号来源**（ADR-0001 / ADR-0003）：买卖信号仅来自 **策略 DSL + 指标引擎**；AI 只做解释、一致性摘要、优化建议文案；**禁止**直接输出荐股或实时买卖点。
3. **优化建议**：仅限规则简化、DSL 参数调整、样本外提示、专项训练编排；任何参数变更须可回测复现。

### 历史 K 线标注的定位

- **非主路径**：降为可选能力（对照、补录、运营），**不得**作为生成主策略的唯一输入。
- 实现已存在的 `annotation-service` 可保留，文档与 H5 入口标为「高级 / 对照」。

## 一致性评估（产品必选能力）

对同一用户多轮盲测，至少输出：

| 分项 | 用途 |
| --- | --- |
| 入场一致性 | 买入时行情特征是否稳定 |
| 出场一致性 | 持仓/止盈/止损模式是否混杂 |
| 节奏一致性 | 交易频率、空仓习惯 |
| 跨轮趋势 | 训练后是否收敛 |

- 一致性**不足**：仅报告 + 训练建议，**不**强行生成 DSL。
- 一致性**达阈值**：提供行为策略 DSL 草稿，**用户确认**后进入回测。

允许用户声明「多行情多套规则」（P2）；默认引导先收敛为一种主风格。

## 执行偏差诊断的演进

| 阶段 | 输入 |
| --- | --- |
| 无行为 DSL | 仅多轮 blind → 一致性报告 |
| 已有行为 DSL | 新一轮 blind vs DSL 信号 → 执行率、收益损耗等（原诊断指标） |

## 影响

- 产品文档、迭代计划、README 定位以对齐本 ADR 为准。
- 新增/扩展 `blind-replay-service`（或等价模块）、一致性用例、行为策略生成用例。
- [02 理想策略生成](../product/modules/02-ideal-strategy-generation.md) 文档语义调整为「行为策略生成」，输入以 blind 为主。
- 小程序 / H5 IA：工作台默认进入「盲测训练」，而非「开卷标注」。

## 备选方案（未采纳为主路径）

- **方案 B（双轨并列）**：开卷「目标策略」+ 闭卷「行为策略」双主线——复杂度高，作为 P2 对照能力保留。
- **原标注优先路径**：见 ADR-0002 历史版本，已让位于本 ADR 的产品优先级。
