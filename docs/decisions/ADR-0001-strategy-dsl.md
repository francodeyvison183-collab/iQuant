# ADR-0001 使用策略 DSL 作为核心契约

## 背景

产品需要将用户的理想买卖点转化为可解释、可回测、可诊断、可预警的交易策略。

如果直接生成代码，会带来安全、可维护性、版本管理和解释困难。

## 决策

使用结构化策略 DSL 作为核心契约。

策略 DSL 用于：

- 回测验证。
- 盲测信号对齐。
- 策略解释。
- 参数调整。
- 策略版本管理。
- 后续信号预警。

## 影响

- 策略生成模块必须输出 DSL。
- 回测引擎必须执行 DSL；回测、盲测、诊断三个模块共享同一份 DSL 求值器（具体边界见 [ADR-0009](ADR-0009-backtest-engine-boundary.md)）。
- 执行诊断模块必须基于 DSL 生成理论信号。
- AI 只能提出 DSL 修改建议，不能直接绕过 DSL。
