# ADR-0002 历史标注与盲测回放必须分离

> **产品主路径更新**：策略生成与一致性收敛已改为以盲测回放为主，见 [ADR-0011 盲测训练归纳行为策略](ADR-0011-blind-replay-primary-strategy-path.md)。  
> 本文 **「数据与样本分源」仍然有效**；仅「谁生成主策略」的优先级已调整。

## 背景

历史 K 线标注和近端历史盲测回放的用户心态不同。

历史标注中，用户能看到完整后续走势，属于“开卷表达”。  
盲测回放中，用户看不到未来走势，属于“类实战决策”。

## 决策

### 1. 数据与流程分离（不变）

- 两类样本 **禁止** 混合训练或写入同一混淆来源字段。
- 表、API、`source` 字段必须可审计区分：`historical_labeling` vs `blind_replay`。
- 策略归纳与一致性评估 **主消费** `blind_replay`（见 ADR-0011）。
- 开卷标注样本 **不得** 替代 blind 样本生成主策略；可作可选对照（P2）。

### 2. 主策略来源（2026-05-19 调整）

| 能力 | 主输入（ADR-0011） | 辅助输入 |
| --- | --- | --- |
| 行为策略 DSL | 多轮盲测操作 + bar 特征 | — |
| 一致性评估 | 多轮盲测 | — |
| 执行偏差诊断（二期） | 已确认行为 DSL vs 新一轮 blind | 可选：开卷标注作「目标对照」 |
| 开卷标注 | — | 补录、对照、运营 |

## 影响

- 行为策略生成模块 **只消费** blind 样本（及用户确认后的 DSL）。
- 执行诊断模块：无 DSL 时输出一致性报告；有 DSL 后输出信号对齐指标。
- 数据模型必须明确记录样本来源；混用视为缺陷。
